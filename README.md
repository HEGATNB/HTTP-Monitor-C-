# HTTP Monitor

## Описание проекта

**HTTP Monitor** - это десктопное приложение для мониторинга и тестирования HTTP-запросов, разработанное на платформе .NET 10.0 с использованием WPF. Приложение сочетает в себе встроенный HTTP-сервер с поддержкой REST API (GET, POST, PUT, DELETE) и HTTP-клиент для отправки запросов, предоставляя полный контроль над сетевым взаимодействием с логированием и аналитикой в реальном времени.

### Функциональные возможности

- **HTTP-Сервер**:
  - Поддержка методов GET, POST, PUT, DELETE
  - Сохранение сообщений в памяти с выдачей уникальных ID
  - Настраиваемый порт сервера
  - Статистика по запросам (количество, среднее время ответа)
  - График пиковой нагрузки
  - Логирование всех входящих запросов в UI и файл

- **HTTP-Клиент**:
  - Отправка GET, POST, PUT, DELETE запросов
  - Поддержка JSON-тел запросов
  - Форматированный вывод ответов
  - Быстрые примеры для тестирования
  - Работа с внешними API

- **Мониторинг и аналитика**:
  - Отображение статистики в реальном времени
  - Фильтрация логов по методу и статусу
  - Сохранение логов в файл `server_logs.txt`
  - Многопоточная обработка запросов
  - Отслеживание времени работы сервера

## Технологический стек

- **.NET 10.0** - платформа разработки
- **WPF (Windows Presentation Foundation)** - технология создания интерфейса
- **XAML** - язык разметки интерфейса
- **HttpListener** - встроенный HTTP-сервер
- **HttpClient** - HTTP-клиент
- **Newtonsoft.Json** - работа с JSON

## Инструкция по установке и запуску

### Предварительные требования

1. **.NET 10.0 SDK** ([скачать](https://dotnet.microsoft.com/download/dotnet/10.0))
2. **Windows 10/11** (для работы WPF-приложений)

### Проверка установки .NET

Откройте командную строку и выполните:
```bash
dotnet --version
```
Установка и запуск

    Клонируйте репозиторий:

bash
```
git clone https://github.com/yourusername/HttpMonitorApp.git
cd HttpMonitorApp
```

### Авто запуск 
- Скачайте и распакуйте проект

- Запустите run.bat двойным кликом

- При необходимости подтвердите права администратора
### Ручной запуск
Восстановите зависимости:
```
bash

dotnet restore
```
Соберите проект:
```
bash

dotnet build
```
Запустите приложение:
```
bash

dotnet run
```
Настройка прав доступа к порту

При первом запуске может потребоваться разрешение на прослушивание порта. Запустите PowerShell от имени администратора и выполните:
(в данном случае команды для русской версии и адреса 8080)
```
netsh http add urlacl url=http://localhost:8080/ user="Все"
netsh http add urlacl url=http://127.0.0.1:8080/ user="Все"
```
## Использование
Запуск сервера

- Перейдите на вкладку "Сервер"

- Введите порт (по умолчанию 8080)

- Нажмите "Запустить"

Отправка запросов

1. Перейдите на вкладку "Клиент"

2. Введите URL и выберите метод (GET/POST/PUT/DELETE)

3. Для POST/PUT добавьте JSON-тело

4. Нажмите "Отправить запрос"

Примеры запросов

- GET /stats - получить статистику сервера
```
http://localhost:8080/stats
```
- POST / - создать сообщение
```
{ "message": "Привет, мир!" }
```
- PUT /1 - обновить сообщение с ID 1
```
{ "message": "Обновленное сообщение" }
```
